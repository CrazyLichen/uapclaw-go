# 11.14 Web Channel Python 对齐修复设计

## 背景

11.14 Web Channel 实现与 Python 存在 21 处差异，涉及架构分层、消息路由、配置处理、会话管理等多个方面。
本设计逐条列出修复方案，确保 Go 实现一比一复刻 Python 逻辑。

## 修复清单

### 问题 1：`Send()` 中 res 帧处理不对齐

**Python**：构造完整 res 帧 `{"type":"res","id":...,"ok":...,"payload":...}`，ok=false 时从 payload 提取 error/code 到顶层，payload 非 dict 时转 `{"content": str(payload)}`。

**Go 修复**：在 `Send()` 的 res 分支中，构造 `ResFrame` 而非调用 `broadcastEvent`，对齐 Python L319-341 逻辑。

### 问题 2：`extractPureTextPayload` 缺失 cron 字段提取

**Python**：当 event_name=="chat.final" 时提取 `msg.payload["cron"]`（dict 类型）。

**Go 修复**：`extractPureTextPayload(msg, eventName)` 增加 eventName 参数，当 eventName=="chat.final" 时提取 cron dict。

### 问题 3：缺失 `_process_files` 文件下载逻辑

**Python**：`_handle_raw_message` 中调用 `_process_files(params)`，遍历 `params["files"]`，下载 URL 文件到 workspace，写回 `path`。

**Go 修复**：实现 `processFiles(params)` 函数，用 `http.Get` 下载文件到 `workspace.AgentWorkspaceDir()`，写回 `file_info["path"]`。在 `HandleWebSocket` 消息处理前调用。

### 问题 4：`agentClient` 直接持有违反分层架构 + 未就绪断开连接

**Python**：WebChannel 不持有 agentClient，通过 `_register_web_handlers(bind)` 的 `onConnect` 钩子闭包捕获。Agent 未就绪时跳过 ack，不断开连接。

**Go 修复**：
1. 移除 `WebChannel.agentClient` 字段和 `SetAgentClient()` 方法
2. 新增 `connectHooks []ConnectHook` 字段和 `OnConnect(callback)` 方法
3. `RegisterWebHandlers` 中定义 onConnect 钩子并注册（闭包捕获 agentClient 引用）
4. `HandleWebSocket` 中：升级 WS → 加入 clients → 触发 onConnect 钩子 → 进入消息循环

### 问题 5：`MakeNormAndForward` 三层路由逻辑缺失

**Python** 三层路由：
- 非 `ForwardReqMethods` → 不转发，只走本地 handler
- `ForwardReqMethods` 有本地 handler → 转发 + 本地 handler（返回 false）
- `ForwardNoLocalHandlerMethods` → 仅转发，短路本地 handler（返回 true）

**Go 修复**：`MakeNormAndForward` 增加 `forwardMethods` 和 `noLocalHandlerMethods` 参数，对齐三层路由判断。

### 问题 6：connection.ack 走消息管道

随问题 4 一并解决：onConnect 钩子中通过 `channel.Send()` 发 connection.ack，走消息管道。

### 问题 7：`handleChannelGet` 硬编码

**Go 修复**：定义 `WebHandlersBindParams` 结构体（对齐 Python `WebHandlersBindParams`），注入 `ChannelManager`，`handleChannelGet` 从 `ChannelManager` 获取 enabled channels。

### 问题 8：`onMessageCb == nil` 无 fallback

**Python**：fallback 到 `self.bus.publish_user_messages(user_message)`。

**Go 修复**：当 `onMessageCb` 为 nil 时，调用 `channelMgr.DeliverToMessageHandler(userMessage)` 作为 fallback。WebChannel 需持有 channelMgr 引用。

### 问题 9：pure-text 路径缺失 error fallback 提取

**Python**：content 为空且 ok=false 时，从 `payload["error"]` 提取。

**Go 修复**：在 `extractPureTextPayload` 中，当 content 为空且 `msg.OK == false` 时，从 `msg.Payload["error"]` 提取。

### 问题 10：`config.get` 中 api_key/token 无解密

**Python**：通过 `ExtensionRegistry.get_crypto_provider().decrypt(val)` 解密。

**Go 修复**：引入 `CryptoProvider` 接口，`config.get` 时对含 `api_key`/`token` 的键解密，`config.set` 时加密。通过 `WebHandlersBindParams` 注入。

### 问题 11：`buildModelsDefaultsFromFrontend` 缺失 merge + infer_is_default

**Python**：
- `_merge_models_for_replace_all`：用 origin_index 匹配原始 YAML，保留占位符和未暴露字段，仅覆写变化字段
- `_infer_is_default`：确保恰好一个 is_default=true

**Go 修复**：完整实现 `mergeModelsForReplaceAll` + `inferIsDefault`。

### 问题 12：`models.list` 返回格式不对齐

**Python**：展平为前端格式（model_name, api_base, api_key, model_provider, temperature, is_default, alias, origin_index），返回 active_model。

**Go 修复**：`handleModelsList` 展平 + origin_index + active_model + api_key 解密。

### 问题 13：`session.create/delete` 逻辑不对齐

**Go 修复**：
- `session.create`：强制 session_id 非空 + 已存在检查（ALREADY_EXISTS）+ init_metadata
- `session.delete`：先转发 AgentServer + team session 保护（AGENT_UNAVAILABLE）+ 不存在检查（NOT_FOUND）

### 问题 14：`chat.interrupt` handler 硬编码 intent

**Python**：从 params 提取 intent，为空则不设置。

**Go 修复**：`handleChatInterrupt` 从 params["intent"] 提取，非空字符串时设入响应。

### 问题 15：`parseMode` 默认值不一致

**Python**：默认 `Mode.AGENT_PLAN`。

**Go 修复**：`parseMode` 默认返回 `schema.ModeAgentPlan`。

### 问题 16：`BuildUserMessage` metadata 缺少 query

**Python**：`metadata={"query": query, "method": method}`。

**Go 修复**：`BuildUserMessage` 增加 `query` 参数，写入 metadata。

### 问题 17：HandleWebSocket 缺失 URL path 校验 + query 解析

**Python**：校验 `request_path != config.path` → 关闭连接（code=1008）；解析 query 参数。

**Go 修复**：`HandleWebSocket` 中校验 `r.URL.Path`，不匹配返回 HTTP 错误；解析 `r.URL.Query()` 传入 `BuildUserMessage`。

### 问题 18：`config.get` 中 memory_forbidden_description 返回字符串非 dict

**Python**：返回多语言 dict（`{"zh": "...", "en": "..."}`）。

**Go 修复**：用 `getConfigAny` 读取整个 description 节点。

### 问题 19：`config.set` 中 memory_forbidden_description 硬编码 zh

**Python**：动态读取 `preferred_language` 作为 dict key。

**Go 修复**：从 config.yaml 读取 preferred_language（默认 zh），用该值作为 key。

### 问题 20：`flattenTeamConfig` 严重简化

**Python**：~100 行展平逻辑（team 索引、leader/teammate/agent 详情提取、predefined_members 序列化）。

**Go 修复**：完整对齐 Python `_flatten_modes_team_for_config_panel`，一比一复刻。

### 问题 21：`config.save_all` 中 models_count 缺省时不设键

**Python**：始终设 models_count 键（值为 None）。

**Go 修复**：始终设置 `"models_count"` 键，无 models 子载荷时值为 nil。

## 架构变更总结

### WebHandlersBindParams（对齐 Python WebHandlersBindParams）

```go
type WebHandlersBindParams struct {
    Channel        *WebChannel
    AgentClient    *routing.AgentClient     // 闭包捕获，不存到 WebChannel
    MessageHandler MessageHandler           // 闭包捕获，用于 ack 优先路径
    ChannelManager *cm.ChannelManager       // 闭包捕获
    OnConfigSaved  OnConfigSavedFunc
    CryptoProvider CryptoProvider           // 加密/解密
    HeartbeatService any                    // ⤵️ 11.15 心跳服务
    CronController  any                     // ⤵️ 11.16 定时任务控制器
    UpdaterService  any                     // ⤵️ 11.17 更新服务
}
```

> **注意**：RegisterWebHandlers 在 app_gateway.go 中调用（对齐 Python app_gateway.py L994），
> 不在 NewWebChannel 内调用，避免 web 包依赖 routing 包。

### WebChannel 结构变更

```
移除:
  - agentClient *routing.AgentClient
  - SetAgentClient()

新增:
  - connectHooks []ConnectHook
  - OnConnect(callback)
  - channelMgr *cm.ChannelManager  // fallback 路由用
```

### MakeNormAndForward 签名变更

```go
// 旧
func MakeNormAndForward(channelMgr *cm.ChannelManager) NormAndForwardFunc

// 新（对齐 Python _make_norm_and_forward）
func MakeNormAndForward(
    channelMgr *cm.ChannelManager,
    forwardMethods map[string]bool,
    noLocalHandlerMethods map[string]bool,
) NormAndForwardFunc
```

### BuildUserMessage 签名变更

```go
// 旧
func BuildUserMessage(reqID, method string, params map[string]any, sessionID string) *schema.Message

// 新
func BuildUserMessage(reqID, method string, params map[string]any, sessionID string, query map[string][]string) *schema.Message
```
