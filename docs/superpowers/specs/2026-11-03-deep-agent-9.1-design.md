# 9.1 DeepAgent 实现设计

## 概述

实现 `DeepAgent` 结构体，作为面向生产的高级 Agent 封装层。DeepAgent 包装 ReActAgent + 任务循环 + Rails + 技能 + 子 Agent，实现两层架构：外层（DeepAgent）负责多轮任务编排，内层（ReActAgent）负责单轮 Think-Act-Observe 推理。

对应 Python：`openjiuwen/harness/deep_agent.py`（2481 行）

## 在 Agent 会话流程中的位置

```
用户请求 (CLI / API / WebSocket)
  → Gateway (领域6) → Channel (领域7)
  → DeepAgent (9.1) ← 我们在这里
    ├── 外层任务循环 (TaskLoop)
    │   └── ReActAgent (领域8) — 内层推理循环
    ├── Rails (9.11-9.24) — 安全/功能护栏
    ├── Skills (技能系统)
    ├── SubAgents (9.25-9.31) — 子 Agent 管理
    └── SysOperation (9.32-9.37) — 系统操作
```

## 结构体设计

```go
type DeepAgent struct {
    // ─── 身份与共享（与 ReActAgent 共享后两个） ───
    card              *agentschema.AgentCard
    abilityManager    interfaces.AbilityManagerInterface
    callbackManager   *rail.AgentCallbackManager

    // ─── 内层 Agent ───
    reactAgent        *agents.ReActAgent

    // ─── 配置 ───
    deepConfig        *hschema.DeepAgentConfig
    systemPromptBuilder *hprompts.SystemPromptBuilder

    // ─── 任务循环 ───
    loopCoordinator   *taskloop.LoopCoordinator
    loopController    *taskloop.TaskLoopController
    loopSession       sessioninterfaces.SessionFacade
    boundSessionID    string
    taskCompletionRail rail.AgentRail

    // ─── Rails ───
    pendingRails      []rail.AgentRail
    staleRails        []rail.AgentRail
    registeredRails   []rail.AgentRail
    railsMu           sync.Mutex

    // ─── 原子标志 ───
    initialized       atomic.Bool
    invokeActive      atomic.Bool
    autoInvokeScheduled atomic.Bool

    // ─── 流控制 ───
    streamMu          sync.Mutex
    streamCancel      context.CancelFunc

    // ─── 会话工具 ───
    sessionToolkit    *subagent.SessionToolkit

    // ─── Harness Config ───
    pendingHarnessConfigs []string

    // ─── 配置读写锁 ───
    configMu          sync.RWMutex
}
```

## 并发安全策略（最小化锁）

Python 依赖 asyncio 单线程模型，Go 需要显式保护。采用最小化锁策略：

| 保护对象 | 机制 | 保护字段 |
|---------|------|---------|
| `configMu` (RWMutex) | configure() 写锁，invoke/follow_up/steer/abort 读锁 | `deepConfig`, `reactAgent`, `loopCoordinator`, `loopController`, `loopSession`, `boundSessionID`, `systemPromptBuilder`, `taskCompletionRail` |
| `railsMu` (Mutex) | configure + ensureInitialized 互斥 | `pendingRails`, `staleRails`, `registeredRails` |
| `streamMu` (Mutex) | stream + abort 互斥 | `streamCancel` |
| `initialized` (atomic.Bool) | ensureInitialized 守卫 | |
| `invokeActive` (atomic.Bool) | invoke 生命周期 | |
| `autoInvokeScheduled` (atomic.Bool) | auto-invoke 调度守卫 | |

**不需要额外保护的部分**：
- `LoopQueues`（channel 天然安全）
- `LoopCoordinator`（内部已有 sync.Mutex）
- `TaskLoopController`（委托给已安全的子组件）
- `follow_up/steer/abort`（走 channel 或已加锁的 LoopCoordinator）

## 核心方法实现

### Configure（首次 + 热重配置）

```
Configure(ctx, config)
  ├── filterDisabledTools(config)
  ├── 首次? → initialConfigure(ctx, config)
  │     ├── deepConfig = config
  │     ├── card = config.Card
  │     ├── reactAgent = createReactAgent()   // 创建内层 Agent
  │     └── queuePendingRails(config)         // 排队 Rails
  └── 后续? → hotReconfigure(ctx, config)
        ├── deepConfig = config
        ├── hotReloadRails(config)            // 旧 Rails → staleRails
        ├── hotReloadModel(ctx, config)       // 热更新 ReActAgent 模型
        ├── hotReloadTools(ctx, config, prev) // 同步 AbilityManager 工具
        ├── hotReloadSystemPrompt(config)     // 重建 SystemPromptBuilder
        ├── queuePendingRails(config)
        └── syncBuilderToActiveRails()
  └── initialized = false   // 强制下次 ensureInitialized
```

### createReactAgent（创建内层 Agent）

```
createReactAgent()
  ├── 创建 innerCard: name = "{card.Name}_react"
  ├── 构建 ReActAgentConfig
  │     ├── maxIterations = MaxInt (任务循环) 或 config.MaxIterations
  │     ├── ContextEngineConfig / Workspace / Model
  │     └── SystemPromptBuilder(language, mode)
  ├── agent = NewReActAgent(innerCard, reactConfig)
  ├── agent.Configure(ctx, reactConfig)     // 内部会覆盖 promptBuilder
  ├── agent.SetPromptBuilder(builder)       // 覆盖回共享 builder
  ├── agent.SetAbilityManager(abilityManager) // 注入共享 AbilityManager
  └── agent.SetLLM(model)                  // 注入预构建 LLM（如果有）
```

### Invoke / Stream（核心执行路径）

```
Invoke(ctx, inputs, opts...)
  ├── ensureInitialized(ctx)
  ├── normalizeInputs(inputs) → InvokeInputs
  ├── 创建 AgentCallbackContext
  ├── invokeActive = true
  ├── lifecycle(BEFORE_INVOKE, AFTER_INVOKE):
  │     ├── 启用任务循环 AND 非恢复输入?
  │     │     └── runTaskLoopInvoke(ctx, session)
  │     └── 否则
  │           └── runSingleRoundInvoke(ctx, inputs, session)
  └── invokeActive = false

Stream(ctx, inputs, opts...)
  ├── ensureInitialized(ctx)
  ├── drainPendingHarnessConfigs(ctx)
  ├── normalizeInputs(inputs) → InvokeInputs
  ├── invokeActive = true
  ├── lifecycle(BEFORE_INVOKE, AFTER_INVOKE):
  │     ├── 启用任务循环 AND 非恢复输入?
  │     │     └── runTaskLoopStream(ctx, session, modes)
  │     │           ├── 后台 goroutine: runTaskLoop → 每轮 reactAgent.Invoke
  │     │           └── 前台: session.StreamIterator() → 转发到 outCh
  │     └── 否则
  │           └── runSingleRoundStream(ctx, inputs, session, modes)
  │                 └── 直接返回 reactAgent.Stream() 的 channel
  └── invokeActive = false
```

**Stream 数据流**：
- 任务循环模式：后台 goroutine 调 `reactAgent.Invoke()`（只写 session stream），前台从 `session.StreamIterator()` 读取（唯一消费者），转发到 DeepAgent 的 `outCh`
- 单轮模式：直接返回 `reactAgent.Stream()` 的 channel
