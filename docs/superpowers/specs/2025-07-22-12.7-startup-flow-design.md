# 12.7 统一启动器启动流程补齐设计

> 对齐 Python：`jiuwenswarm/app.py` + `jiuwenswarm/gateway/app_gateway.py` + `jiuwenswarm/server/app_agentserver.py`

## 1. 分层原则

| 层 | 职责 | 对应 Python |
|----|------|------------|
| **cmd 层（runAppCmd）** | 公共初始化：workspace、logger、dotenv、config、扩展系统、配置热重载 reloader、`UAPCLAW_START_CMD` | app.py 中的公共步骤 |
| **GatewayServer 层** | Gateway 专属启动：配置快照缓存+推送、多路径 WS 路由、IM Pipeline、Cron、Heartbeat | app_gateway.py `_run()` |
| **AgentServer 层** | AgentServer 专属启动：接收配置、reload_agents_config | app_agentserver.py `_run()` |

## 3. GatewayServer 层补齐步骤

### 3.1 配置快照本地缓存 + 启动时推给 AgentServer

**处理方式**：实现

单体架构中 Gateway 和 AgentServer 是独立组件，AgentServer 不共享 Gateway 的 Config 指针。
配置始终通过 `agent.reload_config` E2A 请求传递，机制与 Python WS 通知一致。

**启动时**：
1. Gateway 缓存配置快照（对齐 Python `set_or_update_server_config`）
2. 构造 `agent.reload_config` E2A 请求，params 包含 `{config: fullCfg, env: envMap}`
3. 通过 ChannelTransport 发给 AgentServer

**热重载时**（reloader OnReload 回调）：
1. Gateway 更新本地缓存
2. 构造 `agent.reload_config` E2A 请求，params 包含增量配置和 env 变更
3. 通过 ChannelTransport 发给 AgentServer

**对应 Python**：
- `set_or_update_server_config()` — 本地缓存（agent_client.py L133）
- `agent.reload_config` E2A 请求 — 发给 AgentServer（app_gateway.py L946-960）

### 3.2 多路径 WS 路由框架

**处理方式**：预留 RouteBinding 接口，当前注册 `/ws`(web)

```go
// RouteBinding 描述一个 WS 路径的完整消息处理策略。
// 对齐 Python: GatewayRouteBinding (routing/route_binding.py)
type RouteBinding struct {
    Path                    string           // WS 路径，如 "/ws"、"/acp"、"/tui"
    ChannelID               string           // 通道标识，如 "web"、"acp"、"tui"
    ForwardMethods          map[string]bool  // 需要转发到 AgentServer 的方法集合
    ForwardNoLocalHandler   map[string]bool  // 转发后不需要本地 ack 的方法集合
    InboundInterceptor      InterceptorFunc  // 入站拦截器（如 ACP JSON-RPC 翻译）
    OutboundInterceptor     InterceptorFunc  // 出站拦截器
    DisconnectHandler       DisconnectFunc   // 连接断开回调
    Install                 InstallFunc      // 在 GatewayServer 上注册本地 handler 的钩子
}
```

当前只注册 `/ws` (web channel)，后续 ACP/TUI 实现时注册新路径。

**对应 Python**：`GatewayRouteBinding` + `_build_acp_route_binding` + `build_cli_route_binding`
