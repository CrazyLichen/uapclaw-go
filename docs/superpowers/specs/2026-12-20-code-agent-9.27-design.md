# 9.27 CodeAgent 实现设计

## 概述

实现 CodeAgent（编码子 Agent），对齐 Python `openjiuwen/harness/subagents/code_agent.py`。
CodeAgent 是 DeepAgent 的核心执行引擎，负责将任务落到可运行的代码与可验证的结果。

## 流程位置

```
用户请求 → Gateway → E2A → AgentServer
  → DeepAgent (9.1)
    → TaskLoopController → 子 Agent 分派
      ├─ ResearchAgent (9.25) ✅
      ├─ BrowserAgent (9.26) ✅
      ├─ CodeAgent (9.27) ← 本次实现
      ├─ PlanAgent (9.28) ✅
      └─ ExploreAgent (9.30) ✅
```

## 作用

- 工具驱动编码：通过 read/write/edit/grep/list/bash/code 等工具操作代码库
- 变更可控：变更小、可回滚；先澄清数据结构与接口，再动代码
- 自带子 Agent：自动注入 ExploreAgent + PlanAgent（`_inject_builtin_plan_agents`）
- 强制 Rails：SysOperationRail + AgentModeRail + AskUserRail + ConfirmInterruptRail（`_merge_rails_with_required`）
- 可选 CodingMemoryRail：⤵️ 9.19-23 回填，当前条件守卫跳过
- 启用规划模式：`EnableTaskPlanning=true`

### 步骤 1：重写 code_agent.go — BuildCodeAgentConfig

**文件**：`internal/agentcore/harness/subagents/code_agent.go`

替换旧骨架代码（`map[string]any` 签名 + ⤵️占位），对齐 PlanAgent/ExploreAgent 模式：

- 常量 `CodeAgentFactoryName = "code_agent"`
- 全局变量 `defaultCodeAgentSystemPrompt`（中英双语，一比一复刻 Python）
- 全局变量 `defaultCodeAgentDescription`（中英双语，一比一复刻 Python）
- `BuildCodeAgentConfig(model *llm.Model, params *hschema.SubagentCreateParams) *hschema.SubAgentConfig` — 新签名
  - AgentCard：用户未提供时使用默认
  - SystemPrompt：用户未提供时使用默认
  - 完整参数映射（Tools/ToolInstances/Mcps/Rails/Skills/Backend/Workspace/SysOperation/Language/PromptMode/EnableTaskLoop）
  - MaxIterations：0 时默认 15（对齐 Python `max_iterations=15`）
  - FactoryName = "code_agent"
  - EnablePlanMode = params.EnablePlanMode
  - RestrictToWorkDir：默认 false（对齐 Python `create_code_agent` 不传 `restrict_to_work_dir`，但 code_agent 语义上不限制）
  - FactoryKwargs：⤵️ 9.19-23 CodingMemoryRail 的 embedding_config 回填占位
- `DefaultCodeAgentSystemPrompt(language)` 辅助函数
- `DefaultCodeAgentDescription(language)` 辅助函数

### 步骤 2：新建 code_agent_factory.go — CreateCodeAgent

**文件**：`internal/agentcore/harness/code_agent_factory.go`

一比一对齐 Python `create_code_agent`，4 步核心逻辑：

#### 2a：injectBuiltinPlanAgents

检查 params.Subagents 中是否已有 `explore_agent` / `plan_agent`（通过 `SubagentSpec.SpecName()` 匹配），
没有则自动注入 `BuildExploreAgentConfig` / `BuildPlanAgentConfig` 构建的 `SubAgentConfig`。

对齐 Python `_inject_builtin_plan_agents(subagents, resolved_language=..., model=...)`。

注入的子 Agent 配置：
- explore_agent：`BuildExploreAgentConfig(model, &SubagentCreateParams{Language: language, MaxIterations: 25})`
- plan_agent：`BuildPlanAgentConfig(model, &SubagentCreateParams{Language: language, MaxIterations: 25})`

#### 2b：mergeRailsWithRequired

合并用户 Rails 与必需 Rails，按类型去重（`reflect.TypeOf` 或接口类型断言）。

必需 Rails 列表（对齐 Python `_merge_rails_with_required`）：
1. `rails.NewSysOperationRail()` — 类型 `*rails.SysOperationRail`
2. `rails.NewAgentModeRail()` — 类型 `*rails.AgentModeRail`
3. `interrupt.NewAskUserRail()` — 类型 `*interrupt.AskUserRail`
4. `interrupt.NewConfirmInterruptRail("switch_mode")` — 类型 `*interrupt.ConfirmInterruptRail`

去重规则：遍历必需 Rails 列表，如果用户 Rails 中已有该类型的实例则跳过，否则追加。

#### 2c：CodingMemoryRail 条件守卫

当 params 中提供 EmbeddingConfig 时注入 CodingMemoryRail，否则跳过。
当前 EmbeddingConfig 字段尚未加入 SubagentCreateParams，标注 `⤵️ 9.19-23 CodingMemoryRail 回填`。

#### 2d：调用 CreateDeepAgent

参数映射：
- `EnableTaskPlanning: true`（关键区别，对齐 Python `enable_task_planning=True`）
- `RestrictToWorkDir: false`（对齐 Python 不传 restrict_to_work_dir，但 code_agent 语义上不限制）
- 其他字段：标准映射
