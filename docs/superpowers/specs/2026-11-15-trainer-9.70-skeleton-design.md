# 9.70 Trainer 骨架设计

## 概述

实现 9.70 Trainer（训练器）的纯接口桩骨架。Trainer 是离线自演化的编排器，协调 "evaluate → update → writeback" 循环。

本次只创建结构体字段 + 方法签名（桩实现返回零值或 `errors.New("not implemented")`），所有依赖类型用 `any` 占位，不创建依赖包。

## 在 Agent 会话流程中的位置

```
用户输入 → BaseAgent.Invoke() → Session → ReAct 循环(think→act→observe)
  ↓ 循环结束
  ↓ 产出 Session（含 tracer 轨迹）

离线自演化（Trainer 编排）：
  Trainer.train(agent, train_cases, val_cases)
    → forward: 推理 + 评估 + 从 Session 提取 Trajectory
    → updater.update(trajectories, evaluated_cases) → Updates
    → apply_updates(operators, updates) → 写回 Operator 参数
    → evaluate 验证 → checkpoint → early_stop 判断
    → 返回优化后的 agent

在线自演化（9.24 EvolutionRail 编排，非 Trainer 职责）：
  after_invoke → SignalDetector → OnlineEvolutionOrchestrator → SkillExperienceOptimizer
    → ExperienceManager(stage/approve/reject) → SKILL.md 持久化
```

## 类型设计

### trainer.go

```go
// Trainer 离线自演化训练编排器。
//
// 编排 "evaluate → update → writeback" 自演化循环，
// 接受 Updater 和 BaseEvaluator，管理检查点保存/恢复和早停。
//
// 对应 Python: openjiuwen/agent_evolving/trainer/trainer.py
type Trainer struct {
    // updater 更新生成器（依赖 9.70c Updater Protocol，暂用 any 占位）
    updater any
    // evaluator 评估器（依赖 9.71 BaseEvaluator，暂用 any 占位）
    evaluator any
    // extractor 轨迹提取器（依赖 9.77 Trajectory，暂用 any 占位）
    extractor any
    // callbacks 训练生命周期回调
    callbacks *Callbacks
    // numParallel 并发推理数
    numParallel int
    // earlyStopScore 早停分数阈值
    earlyStopScore float64
    // checkpointStore 检查点存储（依赖 9.78 EvolveCheckpoint，暂用 any 占位）
    checkpointStore any
    // resumeFrom 恢复检查点路径
    resumeFrom string
    // checkpointManager 检查点管理器（依赖 9.78，暂用 any 占位）
    checkpointManager any
}
```

### 方法签名（全部桩实现）

| 方法 | Python 对应 | 签名 | 桩返回 |
|------|------------|------|--------|
| `NewTrainer` | `__init__` | `(opts ...TrainerOption) *Trainer` | 构造函数，正常实现 |
| `Train` | `train()` | `(ctx, agent, trainCases, valCases any, numIterations int) (any, error)` | `nil, errors.New("not implemented")` |
| `Forward` | `forward()` | `(ctx, agent, cases any) (float64, any, any, any, error)` | `0, nil, nil, nil, errors.New("not implemented")` |
| `Evaluate` | `evaluate()` | `(ctx, agent, cases any) (float64, any, error)` | `0, nil, errors.New("not implemented")` |
| `PredictOnly` | `predict_only()` | `(ctx, agent, cases any) (any, error)` | `nil, errors.New("not implemented")` |
| `Predict` | `predict()` | `(ctx, agent, cases any) (any, any, error)` | `nil, nil, errors.New("not implemented")` |
| `ApplyUpdates` | `apply_updates()` | `(operators, updates any)` | 空方法体 |
| `SetCallbacks` | `set_callbacks()` | `(callbacks *Callbacks)` | 赋值，正常实现 |

TrainerOption 函数选项模式：`WithUpdater`、`WithEvaluator`、`WithExtractor`、`WithCallbacks`、`WithNumParallel`、`WithEarlyStopScore`、`WithCheckpointDir`、`WithResumeFrom`、`WithCheckpointManager`。

### progress.go

```go
// Progress 训练进度追踪。
// 对应 Python: openjiuwen/agent_evolving/trainer/progress.py Progress
type Progress struct {
    StartEpoch        int
    CurrentEpoch      int
    MaxEpoch          int
    CurrentBatchIter  int
    MaxBatchIter      int
    BestScore         float64
    BestBatchScore    float64
    CurrentEpochScore float64
}

// Callbacks 训练生命周期回调。
// 对应 Python: openjiuwen/agent_evolving/trainer/progress.py Callbacks
type Callbacks struct {
    // OnTrainBegin 训练开始回调（验证基线评估完成后）
    OnTrainBegin any
    // OnTrainEnd 训练结束回调
    OnTrainEnd any
    // OnTrainEpochBegin 单 epoch 训练开始回调
    OnTrainEpochBegin any
    // OnTrainEpochEnd 单 epoch 训练结束回调
    OnTrainEpochEnd any
}
```
