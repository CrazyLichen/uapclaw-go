# 9.7 循环依赖与接口设计修复方案

> 日期：2026-07-05
> 范围：harness/task_loop + harness/tools/subagent + harness/interfaces + harness/schema + controller + foundation/tool

## 背景

9.7 SessionSpawnExecutor 实现过程中，因 `DeepAgentProvider` 定义在 `task_loop` 包内，
导致 `subagent` 包无法 import `task_loop`（循环依赖），被迫在 subagent 内自建
`SessionToolProvider` 最小接口。同时发现多处实现与 Python 不一致的问题。

本方案将 `DeepAgentProvider`（改名 `DeepAgentInterface`）及相关接口提取到
`harness/interfaces/` 子包，从根源消除循环依赖，并对齐 Python 的访问路径。

## 问题 1：sessionToolkit any → 具体类型

**结论**：无需修复。9.6 中 `sessionToolkit any` 是占位符，9.7 已回填为
`*subagent.SessionToolkit`，当前代码正确。

## 问题 2：循环依赖 → DeepAgentInterface 移到 harness/interfaces/

### 根因

`DeepAgentProvider` 定义在 `task_loop/executor.go`，`subagent` 需要 DeepAgent 能力
但无法 import task_loop（task_loop 也 import subagent 的 SessionToolkit）。

### 修复方案

1. 新建 `harness/interfaces/` 子包
2. 将 `DeepAgentProvider` 改名为 `DeepAgentInterface`，定义在 `harness/interfaces/deep_agent.go`
3. 新建 `LoopCoordinatorInterface`（最小集：`Iteration()` + `RequestAbort()`），
   定义在同一文件
4. `DeepAgentInterface.LoopCoordinator()` 返回 `LoopCoordinatorInterface`
5. `DeepAgentInterface.LoopController()` 返回 `controller.ControllerInterface`
6. `DeepAgentInterface.CreateSubagent()` 返回 `(DeepAgentInterface, error)`
7. 删除 `task_loop/executor.go` 中的 `DeepAgentProvider` 接口
8. 删除 `subagent/session_tools.go` 中的 `SessionToolProvider` 接口
9. 所有消费者改为 import `harness/interfaces`，使用 `DeepAgentInterface`

### 依赖关系（修复后）

```
harness/interfaces/          定义 DeepAgentInterface + LoopCoordinatorInterface
     ↑           ↑
task_loop/      subagent/
(LoopCoordinator 满足 LoopCoordinatorInterface)
(TaskLoopController 满足 ControllerInterface)
```

无循环依赖。

### 对齐 Python

Python 中 `DeepAgent` 定义在 `harness/deep_agent.py`，`task_loop` 和 `subagent` 都
通过 `TYPE_CHECKING` 延迟 import。Go 用 `harness/interfaces/` 接口包达到等价效果。

## 问题 3：ControllerInterface 补方法 + 对齐 Python 取消/提交路径

### 根因

1. `ControllerInterface` 缺少 `TaskScheduler()` 和 `TaskManager()` 方法
2. Go 的 `SessionsCancelTool` 通过 `EventHandler().GetBase().TaskScheduler` 访问，
   Python 通过 `loop_controller.task_scheduler` 访问，路径不一致
3. Go 的 `SessionsSpawnTool` 通过 `EventHandler().GetBase().TaskManager` 访问，
   Python 通过 `loop_controller.event_handler.task_manager` 访问

### 修复方案

1. `controller/interface.go` 的 `ControllerInterface` 增加：
   - `TaskScheduler() *modules.TaskScheduler`
   - `TaskManager() *modules.TaskManager`

2. `SessionsCancelTool.Invoke` 取消路径对齐 Python：
   ```go
   // 修改前（Go 自创路径）
   handler := t.provider.EventHandler()
   scheduler := handler.GetBase().TaskScheduler

   // 修改后（对齐 Python: loop_controller.task_scheduler）
   controller := t.provider.LoopController()
   scheduler := controller.TaskScheduler()
   ```

3. `SessionsSpawnTool.Invoke` 提交路径对齐 Python：
   ```go
   // 修改前（Go 自创路径）
   handler := t.provider.EventHandler()
   tm := handler.GetBase().TaskManager

   // 修改后（对齐 Python: loop_controller → task_manager）
   controller := t.provider.LoopController()
   tm := controller.TaskManager()
   ```

### Python 参考路径

```python
# session_tools.py L182-189
controller = self._parent_agent.loop_controller  # TaskLoopController
scheduler = controller.task_scheduler             # Controller.task_scheduler
success = await scheduler.cancel_task(task_id)
```

## 问题 4：常量移到 harness/schema/task_type.go

### 根因

`SessionSpawnTaskType` 在 `task_loop/executor.go` 和 `subagent/session_tools.go`
各定义一份，值相同但无编译期校验。

### 修复方案

1. 新建 `harness/schema/task_type.go`，定义：
   ```go
   const (
       DeepTaskType         = "deep_agent_task"
       SessionSpawnTaskType = "session_spawn_task"
   )
   ```

2. 删除 `task_loop/executor.go` 中的 `DeepTaskType` + `SessionSpawnTaskType`
3. 删除 `subagent/session_tools.go` 中的 `SessionSpawnTaskType`
4. 两包改为 `import hschema` 引用 `hschema.DeepTaskType` / `hschema.SessionSpawnTaskType`

## 问题 5：NewToolCallOptions nil 防御 + 批量处理

### 根因

1. `NewToolCallOptions` 内部 `for _, opt := range opts { opt(o) }` 在 opt 为 nil 时 panic
2. `SessionsSpawnTool.Invoke` 逐个 opt 循环创建 ToolCallOptions，应改为批量处理

### 修复方案

1. `foundation/tool/base.go` 的 `NewToolCallOptions` 加 nil 防御：
   ```go
   func NewToolCallOptions(opts ...ToolOption) *ToolCallOptions {
       o := &ToolCallOptions{RaiseForStatus: true}
       for _, opt := range opts {
           if opt != nil {
               opt(o)
           }
       }
       return o
   }
   ```

2. `subagent/session_tools.go` SessionsSpawnTool.Invoke 改为批量处理：
   ```go
   // 修改前（逐个循环，错误写法）
   var session any
   for _, opt := range opts {
       if opt != nil {
           o := tool.NewToolCallOptions(opt)
           if o.Session != nil {
               session = o.Session
           }
       }
   }

   // 修改后（批量处理，对齐 StreamFunction/MCPTool 用法）
   callOpts := tool.NewToolCallOptions(opts...)
   session := callOpts.Session
   ```
