# 9.56 Blueprint（团队蓝图定义）完整实现设计

## 概述

9.56 Blueprint 涉及两层实现：
1. **agent/blueprint.go** — TeamAgentBlueprint 不可变蓝图（四象限第一象限）
2. **schema/blueprint.go** — TeamAgentSpec 规格定义 + Build + 校验 + 注册表

完整对齐 Python `openjiuwen/agent_teams/agent/blueprint.py` + `openjiuwen/agent_teams/schema/blueprint.py`，
前置实现所有依赖类型的 Schema（数据结构+JSON标签+校验），运行时方法留桩并标注回填章节。

## 1. 在 Agent 会话中的流程位置与作用

### 流程位置

```
用户调用 TeamAgentSpec.Build()
  │
  ├─ Validate() 校验入口
  │   ├─ validatePoolRouterExclusive()
  │   ├─ defaultTransportForSpawnMode()
  │   ├─ validateReservedNames()
  │   └─ validateHittConsistency()
  │
  ├─ resolveLanguage()          ← ⤵️ 回填: 9.51
  ├─ 构建 TeamSpec
  ├─ 构建 ModelAllocator        ← ⤵️ 回填: 9.64
  ├─ validateLeaderModelResolved()
  ├─ 构建 TeamRuntimeContext
  ├─ 构建 MessagerConfig / DBConfig
  │
  ├─ NewTeamAgent(card)         ← ⤵️ 回填: 9.57
  ├─ agent.AttachModelAllocator()
  └─ agent.Configure(spec, ctx) ← ⤵️ 回填: 9.57
       │
       └─ → 生成 TeamAgentBlueprint ← 9.56 的产出物
```

### 作用

TeamAgentBlueprint 是 TeamAgent 四象限分解的**第一象限（静态数据）**：

| 象限 | 类型 | 可变性 | 章节 |
|------|------|--------|------|
| 1. 静态蓝图 | TeamAgentBlueprint | 构造时确定，不可变 | **9.56** |
| 2. 可变状态 | TeamAgentState | 运行时可变 | 9.55 |
| 3. 进程级基础设施 | TeamInfra | 每进程一份 | 9.55 |
| 4. 实例级资源 | PrivateAgentResources | 每实例一份 | 9.55 |

核心职责：
1. **冻结快照**：将 TeamAgentSpec + TeamRuntimeContext + AgentCard 凝冻为不可变只读对象
2. **访问门面**：通过 Role()/MemberName()/Lifecycle()/TeamSpec() 提供快捷访问
3. **生命周期锚定**：所有 Manager 通过引用 Blueprint 获取不变配置

## 8. 编码规范遵循

- 所有注释使用中文
- 声明排列：结构体 → 枚举 → 常量 → 全局变量 → 导出函数 → 非导出函数
- 分隔注释：`// ──────────────────────────── 结构体 ────────────────────────────`
- 每个 Go 包必须有 `doc.go`
- 日志对齐 Python：校验失败/注册表查找失败等关键路径添加日志
