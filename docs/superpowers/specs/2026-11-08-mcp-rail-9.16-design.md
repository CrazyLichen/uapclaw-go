# 9.16 McpRail 实现设计

## 概述

McpRail 是 DeepAgent 功能 Rail 体系中的一员，职责是在 Agent 初始化时注册两个 MCP 资源浏览工具（ListMcpResourcesTool + ReadMcpResourceTool），使 LLM 能发现和读取已注册 MCP 服务器上的资源。MCP 服务器本身的注册由 DeepAgentConfig.mcps 处理，McpRail 不参与。

## 在 Agent 会话中的流程位置

```
DeepAgent._ensure_initialized()
│
├─ _register_pending_mcps()         ← 先于所有 Rail
│   ├─ ResourceMgr.AddMcpServer()   → 连接 MCP Server + 注册所有 MCPTool
│   └─ AbilityManager.Add(mcpConfig)
│
├─ Rail 初始化（按 priority 高→低）
│   ...
│   McpRail.Init (priority=95)      ← 注册浏览工具到 ResourceMgr + AbilityManager
│   ProgressiveToolRail (90)
│   TaskPlanningRail (90)
│   AgentModeRail (85)
│   HeartbeatRail (80)
│   ...
│
└─ ReAct 循环
    ├─ AbilityManager.ListToolInfo() → MCP 懒加载合成工具名
    ├─ LLM 选择工具
    └─ AbilityManager.Execute() → MCP 工具走 tools 路径
```

## A. harness/tools/mcp/ — MCP 资源浏览工具

### 目录结构

### NewListMcpResourcesTool

- 建卡：`BuildToolCard("list_mcp_resources", "ListMcpResourcesTool", language, nil, agentID)`
- 输入结构体：`{ server_id string }`
- 业务逻辑：`runner.GetResourceMgr().ListMcpResources(ctx, serverID)`
- 返回：`map[string]any{"success": true, "data": [...]}`（uri/name/mimeType/description 字段）
- 错误：`map[string]any{"success": false, "error": err.Error()}`
- 对齐 Python：`openjiuwen/harness/tools/mcp_tools.py ListMcpResourcesTool`

### NewReadMcpResourceTool

- 建卡：`BuildToolCard("read_mcp_resource", "ReadMcpResourceTool", language, nil, agentID)`
- 输入结构体：`{ server_id string, uri string }`
- 业务逻辑：`runner.GetResourceMgr().ReadMcpResource(ctx, serverID, uri)`
- 返回：`map[string]any{"success": true, "data": [...]}`（uri/mimeType/text 字段）
- 错误：同上
- 对齐 Python：`openjiuwen/harness/tools/mcp_tools.py ReadMcpResourceTool`

## B. McpRail

### 结构

```go
type McpRail struct {
    DeepAgentRail
    tools []tool.Tool
}
```

### 方法

- `NewMcpRail() *McpRail` → priority=95，对齐 Python `McpRail.priority = 95`
- `Init(agent)` → 从 agent 获取 language + agentID → 创建两个工具 → `ResourceMgr.AddTool(t)` + `AbilityManager.Add(t.Card())`
- `Uninit(agent)` → `AbilityManager.Remove(name)` + `ResourceMgr.RemoveTool([]string{id})`
- `GetCallbacks()` → 覆盖 Init/Uninit 事件

对齐 Python：`openjiuwen/harness/rails/mcp_rail.py McpRail`

## MCP 工具执行机制说明

MCP 工具的真实执行走 `_tools` 路径而非 `_mcp_servers` 路径：

1. `ListToolInfo()` 被调用时，MCP 懒加载将工具重命名为 `mcp_{serverName}_{toolName}` 写入 `_tools` 字典
2. LLM 返回 `tool_call(name="mcp_filesystem_read_file")` 时，命中 `_tools` 分支
3. 通过 `tool_card.id`（格式 `"serverId.serverName.toolName"`）在 ResourceMgr 找到 MCPTool 实例
4. 调用 `MCPTool.Invoke()` → `McpClient.CallTool()` 转发到 MCP 服务器

`_mcp_servers` 分支是防御性代码（key 是 `serverName` 而非 `mcp_` 前缀工具名），正常不会命中，改为 warning 日志 + fallback。
