> 对齐 Python: `MessageHandler(agent_client)` — 只需 1 个参数

### 1.1 修改 `message_handler.go`

- 删除 `channelMgr *channel_manager.ChannelManager` 字段

### 1.2 修改 `channel_state.go` — resolveControlChannelType 对齐 Python

Python `_resolve_control_channel_type` 实现（L339-345）：
```python
def _resolve_control_channel_type(self, msg):
    provider_raw = getattr(msg, "provider", None)
    provider = str(getattr(provider_raw, "value", provider_raw) or "").strip()
    if provider:
        return provider
    return str(getattr(msg, "channel_id", "") or "")
```

Go 改为：
```go
func (mh *MessageHandler) resolveControlChannelType(msg *schema.Message) channel_manager.ChannelType {
    // 优先 msg.Provider（对齐 Python: msg.provider）
    if msg.Provider != "" {
        return channel_manager.ChannelType(msg.Provider)
    }
    // fallback: msg.ChannelID（对齐 Python: msg.channel_id）
    return channel_manager.ChannelType(msg.ChannelID)
}
```

注意：需要确认 `schema.Message` 是否有 `Provider` 字段，如无则需补充。

## 步骤 2：ChannelManager 合并 MessageHandlerInterface，对齐 Python 3 参数构造

> 对齐 Python: `ChannelManager(message_handler, config=None, on_config_updated=None)` — 3 个参数

### 2.1 新建 `MessageHandlerInterface` 接口

在 `channel_manager.go` 中：

```go
// MessageHandlerInterface 消息处理器接口，由 MessageHandler 实现。
//
// 合并入站处理 + 出站消费为单一接口，对齐 Python ChannelManager(message_handler)。
type MessageHandlerInterface interface {
    // HandleMessage 处理入站消息（对齐 Python MessageHandler.handle_message）
    HandleMessage(msg *schema.Message)
    // ConsumeRobotMessages 从出站队列消费消息（对齐 Python MessageHandler.consume_robot_messages）
    ConsumeRobotMessages(timeout time.Duration) *schema.Message
}
```

## 步骤 3：WebChannel 对齐 Python 两层架构

> Python 两层逻辑（`_handle_raw_message` L505-547）：
> 1. 先调 `_on_message_cb(user_message)` — normalize+forward，返回 bool
> 2. 如果 `handled_by_callback=True` → 短路 return
> 3. 否则走 `_method_handlers[method]` — 本地处理

### 3.4 修改 HandleWebSocket — 增加两层逻辑

对齐 Python `_handle_raw_message`（L457-547）：

```go
// 在消息读取循环中，解析帧后：
// 1. 构造 Message 对象
// 2. 第1层：调 onMessageCb(msg)，返回 true 则短路
// 3. 第2层：走 dispatcher.Dispatch(method, params, sid)
```

具体修改 `HandleWebSocket` 中的消息处理部分：
- 解析请求帧后，构造 `schema.Message` 对象
- 如果 `wc.onMessageCb != nil`，调用 `wc.onMessageCb(msg)`
- onMessageCb 签名改为返回 bool：`func(*schema.Message) bool`
- 如果返回 true，直接 continue（短路，不发响应帧）
- 否则走 `dispatcher.Dispatch(method, params, sid)`

### 4.3 实现 makeNormAndForward

对齐 Python `_make_norm_and_forward`（app_gateway.py L1007-1023）：

```go
// makeNormAndForward 创建 Web 入站 normalize+forward 回调
// 对齐 Python: _make_norm_and_forward (app_gateway.py L1007-1023)
func makeNormAndForward(
    channelMgr *cm.ChannelManager,
    forwardMethods map[string]bool,
    noLocalMethods map[string]bool,
    sourceLabel string,
) func(*schema.Message) bool {
    return func(msg *schema.Message) bool {
        methodVal := string(msg.ReqMethod)
        if !forwardMethods[methodVal] {
            return false
        }
        normalized := normalizeGatewayMessage(msg)
        channelMgr.DeliverToMessageHandler(normalized)
        logger.Info(logComponent).
            Str("source", sourceLabel).
            Str("msg_id", msg.ID).
            Str("channel_id", msg.ChannelID).
            Msg("入站 -> MessageHandler")
        if noLocalMethods[methodVal] {
            return true // 短路，不再走本地 handler
        }
        return false // 继续走本地 handler（如 chat.send 的 ack）
    }
}
```
